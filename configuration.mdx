## Theme and Branding

Tailor your docs appearance starting with the brand color `#3B82F6`.

<Steps>
  <Step title="Access Settings" icon="settings">
    Navigate to your space settings panel.
  </Step>
  <Step title="Update Colors" icon="palette">
    Enter your primary color as `#3B82F6`.
  </Step>
  <Step title="Upload Logo" icon="image">
    Add a logo file under the branding section.
  </Step>
  <Step title="Preview Changes" icon="eye">
    Save and preview the updated theme.
  </Step>
</Steps>

Configure advanced theme options in `aisten.config.json`:

<CodeGroup tabs="JSON,YAML">
```json
{
  "theme": {
    "primaryColor": "#3B82F6",
    "logo": "/path/to/logo.svg",
    "fontFamily": "Inter, sans-serif"
  }
}
```

```yaml
theme:
  primaryColor: "#3B82F6"
  logo: /path/to/logo.svg
  fontFamily: Inter, sans-serif
```
</CodeGroup>

<Callout kind="tip">
  Test your theme across light and dark modes for best results.
</Callout>

## User Roles and Permissions

Define roles to secure your documentation.

| Role       | Can Edit | Can View | Can Admin |
|------------|----------|----------|-----------|
| Viewer     | No       | Yes      | No        |
| Editor     | Yes      | Yes      | No        |
| Admin      | Yes      | Yes      | Yes       |

Assign roles via the members tab. Use `{role: "editor"}` in API calls for programmatic access.

## Export and Import Options

Transfer content seamlessly.

<Tabs>
  <Tab title="Export to MDX" icon="code">
    Select pages and download as MDX files.
    
    Example CLI command:
    ```bash
    aisten export --format=mdx --output=./docs
    ```
  </Tab>
  <Tab title="Import from Markdown" icon="file-text">
    Upload Markdown folders to import.
    
    ```bash
    aisten import --source=./markdown-docs --space=my-space
    ```
  </Tab>
  <Tab title="PDF Export" icon="file">
    Generate PDFs for static sharing.
    
    Set options like `margin: 1in`.
  </Tab>
</Tabs>

## Integrations

Connect aisten with external tools.

<ExpandableGroup>
  <Expandable title="GitHub Sync" default-open="true">
    Link your repo for automatic deployments.
    
    Add webhook: `https://api.aisten.com/webhooks/github/YOUR_WEBHOOK_ID`
    
    <CodeGroup tabs="Env Var,Config">
    ```bash
    AISTEN_GITHUB_TOKEN=your-github-token-here
    ```
    
    ```json
    {
      "integrations": {
        "github": {
          "repo": "your-org/your-repo",
          "token": "YOUR_GITHUB_TOKEN"
        }
      }
    }
    ```
    </CodeGroup>
  </Expandable>
  
  <Expandable title="Slack Notifications">
    Receive updates on doc changes.
    
    Use `https://hooks.slack.com/services/YOUR/SLACK/WEBHOOK` as the endpoint.
  </Expandable>
</ExpandableGroup>

<Callout kind="info">
  Review integration permissions before enabling production use.
</Callout>
